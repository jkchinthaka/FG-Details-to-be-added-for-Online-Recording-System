# Form standards

1. Visible labels — never placeholder-only  
2. Required / optional indicators where helpful  
3. Helper text in plain English  
4. Inline field errors associated with controls (`aria-invalid`, `aria-describedby`)  
5. Form-level `FormErrorSummary` on submit  
6. Focus first invalid / summary after failed submit  
7. Correct `input` types, min/max, date constraints  
8. Character counters where length limits matter  
9. File type/size guidance before upload  
10. Loading disable on submit controls  
11. Keyboard submit supported  
12. Unsaved-change warning for significant drafts  
13. Confirmation after successful save (toast or banner)  
14. Preserve input after recoverable network failure  
15. Business language — never raw “Error 500”

Example recovery copy:

> We could not save this inspection because the server is temporarily unavailable. Your entered data has been preserved. Try again.
