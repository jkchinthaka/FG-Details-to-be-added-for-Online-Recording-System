# UX principles

## Design for the factory floor

The interface is built for Nelna Finished Goods and QA staff working primarily on phones, often with wet or gloved hands and limited time between tasks.

## Principles

1. **Mobile first** — phone layouts are the source of truth; tablet and desktop enhance, they do not redefine.
2. **Minimal clicks** — happy-path records use Mark All Acceptable, then exception edits only.
3. **Minimal typing** — prefer large segmented controls and predefined failure reasons.
4. **Large touch targets** — approximately 44–48px minimum.
5. **Clear task status** — Assigned, In progress, Submitted, Verified, Needs correction.
6. **Cream and white surfaces** — cream page background, white operational cards.
7. **Green primary actions** — `#27743A` for submit and primary CTAs.
8. **Gold as controlled accent** — pending states and important highlights only (`#D8C76B`).
9. **Progressive disclosure** — failure fields appear only after Fail / Unacceptable.
10. **Automatic context capture** — date, time, user and shift without manual re-entry.
11. **Sticky mobile submission** — primary submit actions remain reachable with one thumb.
12. **Autosaved drafts** — protect against interrupted shifts.
13. **Clear error recovery** — validation highlights the failed item, not a vague toast only.
14. **Accessible contrast and keyboard navigation** — visible focus rings; no tiny checkboxes as primary controls.
15. **No horizontal scrolling on mobile**
16. **No oversized dashboard charts** on operator home screens

## Microcopy tone

Use Nelna factory language: Finished Goods, Changing Room, Freezer truck, Corrective action, Verified by. Avoid generic SaaS filler.
